# GuardPilot Evidence Pack / 有效性证据包

This document explains what data is used by GuardPilot and how to verify that the risk gateway has measurable impact in the included deterministic replay.

本文说明 GuardPilot 当前使用的数据类型，并提供一套可复现实验证据，用来证明风控网关在内置场景中确实降低了风险。

---

## Data Truthfulness / 数据真实性说明

GuardPilot does **not** claim to use live exchange execution records in this hackathon MVP.

GuardPilot 当前 Hackathon MVP **不声称使用真实交易所实盘成交记录**。

What is included:

- Deterministic sample market tape: `samples/market/btcusdt_1m.csv`
- Deterministic sample Agent intents/signals: `samples/agents/demo_momentum_signals.jsonl`
- Replay scenario definition: `samples/scenarios/btc_momentum_crash.json`
- Paper-trading output generated from the same inputs every time
- API logs, trade logs, risk events, and risk reports generated by GuardPilot

项目包含的是：

- 确定性样本行情：`samples/market/btcusdt_1m.csv`
- 确定性样本 Agent 交易意图/信号：`samples/agents/demo_momentum_signals.jsonl`
- Replay 场景定义：`samples/scenarios/btc_momentum_crash.json`
- 每次运行都可复现的 paper-trading 输出
- GuardPilot 生成的 API 调用日志、交易日志、风控事件和风险报告

Why this still matters:

- The input stream is fixed and reviewable.
- The baseline and guarded results use the same Agent intents and market tape.
- The only difference is whether GuardPilot blocks/warns high-risk intents.
- Therefore the comparison isolates the value of the guardrail layer.

为什么这仍然能证明项目有效：

- 输入流固定、可检查、可复现。
- baseline 和 with-GuardPilot 使用同一组 Agent 意图与行情。
- 唯一区别是是否经过 GuardPilot 的风控拦截与警告。
- 因此对比结果可以隔离出风控层带来的效果。

## Reproducible Command / 可复现命令

From the repository root:

```bash
npm run replay
npm run evidence
```

Or manually:

```bash
cd guardpilot
python3 scripts/replay.py --scenario samples/scenarios/btc_momentum_crash.json
python3 scripts/verify_evidence.py
```

## What the Evidence Shows / 证据说明

### 1. GuardPilot blocks dangerous behavior

The replay contains 42 Agent trade intents. GuardPilot blocks 22 of them and warns 4 more. The blocked cases include:

- Excessive single-trade notional
- Leverage above the configured maximum
- Overtrading within a short time window
- Consecutive-loss behavior
- Revenge-trading style size escalation
- Low-confidence oversized trades

### 2. GuardPilot reduces downside risk

With the same Agent inputs and market tape:

- Without GuardPilot: max drawdown is `2.60%`
- With GuardPilot: max drawdown is `0.70%`
- Relative drawdown reduction: `73.11%`

This is the main infrastructure value: GuardPilot does not need to predict the market. It reduces the damage caused by unsafe Agent behavior before execution.

### 3. GuardPilot improves the simulated outcome in this scenario

With the same initial balance of `10,000 USDT`:

- Without GuardPilot: final equity is `9,930.81 USDT`
- With GuardPilot: final equity is `9,980.91 USDT`
- Improvement: `+50.10 USDT`

This is not presented as a universal trading-performance claim. It is a deterministic scenario result showing that the guardrail prevents specific harmful behaviors in the sample Agent stream.

### 4. GuardPilot generates audit evidence

The replay generates reviewable artifacts:

- `samples/outputs/sample_api_calls.jsonl`
- `samples/outputs/sample_trade_log.jsonl`
- `samples/outputs/sample_risk_events.jsonl`
- `samples/outputs/sample_risk_report.json`
- `samples/outputs/evidence_manifest.json`
- `reports/demo_report.html`

These files let a reviewer inspect the exact decisions, reasons, risk scores, and simulated execution outcomes.

### 5. GuardPilot makes the evidence tamper-checkable

Each replay writes `samples/outputs/evidence_manifest.json`. The manifest records SHA-256 hashes, byte sizes, and row counts for the scenario file, market data, Agent signal stream, risk profile, API logs, trade logs, risk events, report, and summary.

`npm run evidence` verifies that:

- Manifest hashes match the current files.
- JSONL row counts match the replay totals.
- API log rows equal total intents.
- Trade log rows equal total intents.
- Risk event rows equal `WARN + BLOCK`.
- Summary, report, and manifest agree on the scenario and headline metrics.

This makes the evidence pack stronger than a screenshot: reviewers can rerun the scenario and verify that the artifacts match the reported results.

## Important Limitation / 重要限制

This evidence proves the GuardPilot mechanism works on the included deterministic scenario. It does **not** prove live profitability, and it should not be interpreted as financial advice or as a real exchange trading record.

这套证据证明的是：GuardPilot 在内置确定性场景中可以拦截高风险 Agent 行为、降低模拟回撤并生成审计证据。它 **不证明实盘盈利能力**，也不应被描述为真实交易所成交记录。

## What the Evidence Shows / 证据说明

### 1. GuardPilot blocks dangerous behavior

The replay contains 42 paper-agent trade intents derived from a Bitget public market snapshot. GuardPilot blocks 14 of them. The blocked cases include:

- Excessive single-trade notional
- Leverage above the configured maximum
- Overtrading within a short time window
- Consecutive-loss behavior
- Revenge-trading style size escalation
- Low-confidence oversized trades

### 2. GuardPilot reduces downside risk

With the same Agent inputs and Bitget market tape:

- Without GuardPilot: max drawdown is `1.88%`
- With GuardPilot: max drawdown is `0.27%`
- Relative drawdown reduction: `85.68%`

This is the main infrastructure value: GuardPilot does not need to predict the market. It reduces the damage caused by unsafe Agent behavior before execution.

### 3. GuardPilot improves the simulated outcome in this scenario

With the same initial balance of `10,000 USDT`:

- Without GuardPilot: final equity is `9,828.07 USDT`
- With GuardPilot: final equity is `9,976.93 USDT`
- Improvement: `+148.86 USDT`

This is not presented as a universal trading-performance claim. It is a replay result showing that the guardrail prevents specific harmful behaviors in the paper-agent stream.

### 4. GuardPilot generates audit evidence

The replay generates reviewable artifacts:

- `samples/market/bitget_btcusdt_1m.csv`
- `samples/market/bitget_btcusdt_1m.provenance.json`
- `samples/agents/paper_momentum_agent_signals_from_bitget_snapshot.jsonl`
- `samples/outputs/sample_api_calls.jsonl`
- `samples/outputs/sample_trade_log.jsonl`
- `samples/outputs/sample_risk_events.jsonl`
- `samples/outputs/sample_risk_report.json`
- `samples/outputs/evidence_manifest.json`
- `reports/demo_report.html`

These files let a reviewer inspect the exact market input, decisions, reasons, risk scores, and simulated execution outcomes.

### 5. GuardPilot makes the evidence tamper-checkable

Each replay writes `samples/outputs/evidence_manifest.json`. The manifest records SHA-256 hashes, byte sizes, and row counts for the scenario file, market data, market provenance, Agent signal stream, signal provenance, risk profile, API logs, trade logs, risk events, report, and summary.

`npm run evidence` verifies that:

- Manifest hashes match the current files.
- Market CSV hash matches the provenance file.
- Market source is Bitget public market API.
- JSONL row counts match the replay totals.
- API log rows equal total intents.
- Trade log rows equal total intents.
- Risk event rows equal `WARN + BLOCK`.
- Summary, report, and manifest agree on scenario and headline metrics.
- Execution mode is paper trading only and `live_orders` is false.

This makes the evidence pack stronger than a screenshot: reviewers can rerun the scenario and verify that the artifacts match the reported results.

## Bitget AI Agent Dry-Run Contract Evidence

The replay evidence above proves GuardPilot's risk mechanism. A separate dry-run contract trace proves the Bitget Agent Hub / Playbook / Skill Hub / MCP-style boundary without requiring credentials:

```bash
npm run bitget:trace
```

Generated file:

- `samples/outputs/bitget_agenthub_dry_run_response.json`

What it verifies:

1. A risky Agent Hub / Playbook-style payload with Skill Hub context is normalized into a GuardPilot intent, receives `BLOCK`, and returns `bitget_dry_run: null`.
2. A conservative Agent Hub / Playbook-style payload receives `ALLOW` and returns a Bitget-compatible dry-run execution preview with `guardpilotPolicy: ALLOW_OR_WARN_ONLY_DRY_RUN_NO_LIVE_ORDER`.
3. The trace explicitly states `live_forwarding.enabled: false` and includes the same Bitget public snapshot market context.

This trace is paper/dry-run only. It is designed for judges who need credential-free proof that GuardPilot is not merely a generic dashboard: it is the safety contract before order-capable AI Agent tools.

## Important Limitation / 重要限制

This evidence proves the GuardPilot mechanism works on a replay over real public market candles. It does **not** prove live profitability, and it should not be interpreted as financial advice or as a real exchange trading record.

这套证据证明的是：GuardPilot 在真实公开行情回放中可以拦截高风险 Agent 行为、降低模拟回撤并生成审计证据。它 **不证明实盘盈利能力**，也不应被描述为真实交易所成交记录。
